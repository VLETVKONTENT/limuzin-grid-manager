# v1.5.0 - Многозонный экспорт KML, SVG, GeoJSON и CSV

Дата: 2026-04-08

Статус: стабильная версия, принята пользователем после ручного теста и подготовлена к публикации.

## Что изменено

- Включен пользовательский многозонный экспорт для KML, ZIP, SVG, GeoJSON и CSV.
- Добавлен helper `zone_for_y_interval()` в `core/zones.py`: он определяет зону для интервала `Y` и запрещает только тот случай, когда одна экспортируемая ячейка пересекает внутреннюю границу зоны.
- `core/stats.py` больше не блокирует всю область только из-за нескольких зон, но проверяет выравнивание экспортируемых ячеек `1000x1000` и `100x100` относительно границ зон.
- `GridStats.zone` теперь корректно возвращает одиночную зону по `zone_segments`, даже если правое ребро области лежит ровно на границе следующей зоны.
- KML и ZIP выбирают `pyproj`-трансформер по зоне большого квадрата; `tile_###.kml` внутри ZIP сохраняют прежние имена.
- GeoJSON и CSV пишут `zone` по каждой ячейке и считают WGS84 через трансформер этой зоны.
- SVG пишет список зон в корневой `data-zone` и конкретную зону в `data-zone` каждого прямоугольника и подписи.
- Сводка вкладки `Экспорт` показывает зоны экспорта и сообщает, что трансформер выбирается по ячейке.
- Версия обновлена до `1.5.0` в `pyproject.toml`, `src/limuzin_grid_manager/__init__.py` и `version_info.txt`.
- Обновлены `README.md`, `USER_GUIDE.md`, `GRIDBASE.md`, `ROADMAPv2.md` и индекс версий.

## Оставшиеся ограничения

- Если одна экспортируемая ячейка `1000x1000` или `100x100` сама пересекает внутреннюю миллионную границу зоны, экспорт блокируется. Для такого случая нужно включить округление или выровнять `Y` так, чтобы граница зоны проходила по ребру ячейки.
- Автоматическое разрезание одной ячейки на зональные части не входит в `v1.5.0` и должно быть отдельной задачей, если понадобится.
- GitHub commit, tag, push и Release выполняются после фиксации этой стабильной версии.
