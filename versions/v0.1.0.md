# v0.1.0: движок схем нумерации 100x100

Дата: 2026-04-07

Статус: на тестировании, не коммитить и не пушить до ручной проверки.

## Что добавлено

- Добавлен отдельный движок нумерации малых квадратов `100x100`.
- Старый boolean `snake_small` заменен на расширяемые настройки:
  - схема: `обычная` или `змейка`;
  - направление: `по строкам` или `по колонкам`;
  - стартовый угол: `NW`, `NE`, `SW`, `SE`.
- Дефолт оставлен совместимым с `v0.0.1`:
  - схема `змейка`;
  - направление `по строкам`;
  - старт `NW`.
- Новый движок работает:
  - для `100x100` внутри каждого большого квадрата `1000x1000`;
  - для режима, где включена только сетка `100x100` на всю область.

## Интерфейс

- В блоке `Сетка` вместо чекбокса “Нумерация 100x100 змейкой” добавлены выпадающие списки:
  - `100x100 схема`;
  - `100x100 направление`;
  - `100x100 старт`.
- Левая панель сделана шире и прокручиваемой, а выпадающие списки получили нормальную минимальную ширину, чтобы их было удобно выбирать при тестировании.
- В панели статистики отображается выбранная схема нумерации `100x100`.

## Проверки

- Добавлены unit-тесты для всех комбинаций:
  - схема;
  - направление;
  - стартовый угол.
- Проверяется, что в сетке `10x10` нет пропущенных и повторяющихся номеров.
- Проверяется, что дефолтная змейка из `NW` совпадает со старой логикой `v0.0.1`.

## Что проверить вручную

- Экспорт KML с дефолтной нумерацией: результат должен вести себя как `v0.0.1`.
- Экспорт KML/ZIP со стартом `NE`, `SW`, `SE`.
- Переключение `по строкам` / `по колонкам`.
- Переключение `обычная` / `змейка`.
- Режим только `100x100` без больших квадратов.
