# v2.0.3 - Separate points window inside the app

Дата: 2026-04-10

Статус: принятая версия после ручного пользовательского теста и офлайн-релизной проверки.

## Что изменено

- В [`src/limuzin_grid_manager/ui/points_window.py`](../src/limuzin_grid_manager/ui/points_window.py) добавлено отдельное окно `Точки из Excel`: загрузка `.xlsx`, сводка импорта, preview-таблица точек, блок ошибок по строкам, выбор общего цвета и прозрачности точек и сохранение итогового point-KML.
- В [`src/limuzin_grid_manager/ui/main_window.py`](../src/limuzin_grid_manager/ui/main_window.py) добавлено меню `Инструменты` с действием `Точки из Excel...`; главное окно повторно использует один и тот же экземпляр `PointsWindow` и не смешивает point-flow с grid-вкладками `Предпросмотр`, `Проверка` и `Экспорт`.
- Point-flow по-прежнему изолирован от grid-flow: новые настройки не попадают в `GridOptions`, `ExportMode`, `ProjectState` и `.lgm.json`, а хранятся локально через `QSettings`.
- Обновлены версия приложения и EXE-ресурсов до `2.0.3`.

## Что проверено

- `uv run --extra dev pytest tests\test_points.py tests\test_ui.py`: `15 passed`.
- `uv run --extra dev pytest`: `81 passed`.
- `uv run --extra dev python -m compileall src tests`: компиляция завершилась без ошибок.
- `uv lock --offline`, `uv run --offline --extra dev pytest`, `uv run --offline --extra dev python -m compileall src tests`: офлайн-релизный пайплайн пройден.
- `$env:UV_OFFLINE='1'; .\build_exe_windows.bat`: собран `dist/LIMUZIN_GRID_MANAGER.exe` с версией `2.0.3.0`.
- Пользователь вручную протестировал текущую версию и подтвердил, что внесенные изменения работают.

## Что еще не входит в эту версию

- Отдельный фоновый workflow point-export с прогрессом и отменой.
- Дополнительная релизная полировка point-flow и hardening сборки вокруг Excel-зависимостей.
