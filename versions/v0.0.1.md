# v0.0.1: первая рабочая версия

Дата: 2026-04-07

## Что вошло

- Создано новое Windows-приложение на PySide6.
- Логика координат и сеток вынесена отдельно от интерфейса.
- Добавлен ввод двух точек: `NW` и `SE`.
- Поддержаны координаты СК-42 / Пулково-42, Гаусс-Крюгер, метры.
- Добавлено преобразование координат в WGS84 через `pyproj`.
- Добавлены сетки:
  - `1000x1000`;
  - `100x100`;
  - `100x100` внутри каждого `1000x1000`.
- Добавлена нумерация квадратов змейкой.
- Добавлен экспорт:
  - один общий `.kml`;
  - `.zip` с отдельным `tile_###.kml` на каждый большой квадрат.
- Цветная заливка отключена: KML использует стандартные непрозрачные линии и `<fill>0</fill>`.
- Экспорт выполняется в worker-потоке, чтобы окно не зависало.
- Добавлены базовые тесты ядра и экспорта.
- Добавлена сборка EXE через `build_exe_windows.bat`.
- Добавлен технический файл `GRIDBASE.md` для будущих чатов с Codex.

## Что проверено

- Экспорт KML проверен пользователем: работает.
- `uv run --extra dev pytest` — тесты проходят.
- `uv run --extra dev python -m compileall src tests` — компиляция проходит.
- `build_exe_windows.bat` — EXE собирается.
- `dist/LIMUZIN_GRID_MANAGER.exe` — приложение стартует.

## Известные ограничения

- Многозонные области Гаусса-Крюгера не режутся автоматически.
- Предпросмотра карты или схемы сетки пока нет.
- ZIP-экспорт доступен только при включенной сетке `1000x1000`.
