# v1.2.0 — GeoJSON и CSV

Дата: 2026-04-08

Статус: стабильная версия, принята пользователем после ручного теста и подготовлена к публикации.

## Что изменено

- Добавлены режимы `ExportMode.GEOJSON` и `ExportMode.CSV`.
- Добавлен `src/limuzin_grid_manager/core/geojson.py`: потоковая запись одного `.geojson` как `FeatureCollection` с `Polygon` в WGS84 и свойствами `layer`, `zone`, `cell_size_m`, `big_number`, `big_name`, `small_number` и границами СК-42.
- Добавлен `src/limuzin_grid_manager/core/csv_export.py`: запись одного `.csv` в UTF-8 with BOM с разделителем `;`, границами ячеек в СК-42 и центрами WGS84.
- Реестр `app/export_formats.py` теперь содержит форматы `geojson_gis` и `csv_table`, нормализует расширения `.geojson`/`.csv` и показывает сводку для новых форматов.
- `export_grid()` маршрутизирует GeoJSON и CSV через тот же временный файл и атомарную замену результата, что KML/ZIP/SVG.
- `.lgm.json` сохраняет `export_format_id` для GeoJSON и CSV без изменения версии схемы проекта.
- Вкладка `Экспорт` показывает GeoJSON и CSV, обновляет имена файлов и сводку.
- `.gitignore` дополнен пользовательскими экспортами `*.geojson` и `*.csv`.
- Версия обновлена до `1.2.0` в `pyproject.toml`, `src/limuzin_grid_manager/__init__.py` и `version_info.txt`.
- Обновлены `README.md`, `USER_GUIDE.md`, `GRIDBASE.md`, `ROADMAPv2.md` и индекс версий.

## Оставшиеся ограничения

- Автоматическое разбиение области по зонам Гаусса-Крюгера еще не включено; оно остается для следующих этапов `ROADMAPv2.md`.
- GeoJSON и CSV в `v1.2.0` поддерживают текущую однозонную модель и используют ту же блокировку пересечения зон, что KML/ZIP/SVG.
- DXF, PDF и map-backed preview не входят в объем `v1.2.0`.
- GitHub tag, GitHub Release и публикация не выполнялись.
