# v0.8.5 — Промежуточная стабилизация после ревью

Дата: 2026-04-07

Статус: рабочая версия.

## Что исправлено

- Валидация зон Гаусса-Крюгера перенесена на уровень статистики: зоны вне диапазона `1..32` теперь сразу блокируют экспорт и отображаются как ошибка проверки.
- Загрузка `.lgm.json` больше не использует `bool(...)` для JSON-полей. Значения `include_1000`, `include_100`, `snake_big` и `small_fill_enabled` принимаются только как настоящие JSON `true`/`false`.
- Поврежденные списки проекта, например `big_tile_names` или `custom_big_fill_colors` без поля `number`, теперь дают понятную ошибку открытия проекта вместо внутреннего `KeyError`.

## Что проверено

- Добавлен регрессионный тест на невалидную зону Гаусса-Крюгера.
- Добавлены регрессионные тесты на строковые boolean в `.lgm.json` и поврежденные списки проекта.
- Unit-тесты ядра, экспорта, форматов экспорта, проектов и UI проходят.
- Проверена компиляция `src` и `tests`.
- Собран `dist/LIMUZIN_GRID_MANAGER.exe` с версией `0.8.5.0`.
- Ручной тест версии пользователем пройден.

Проверки:

```powershell
uv run --extra dev pytest
uv run --extra dev python -m compileall src tests
.\build_exe_windows.bat
```

Результат: `42 passed`.
