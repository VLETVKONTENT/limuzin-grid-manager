# LIMUZIN GRID MANAGER

**Текущая версия:** `v0.8.5`
**Платформа:** Windows
**Назначение:** генератор KML-сеток для AlpineQuest

LIMUZIN GRID MANAGER строит сетки `1000x1000` и `100x100` метров по двум точкам в системе СК-42 / Пулково-42, Гаусс-Крюгер, а затем экспортирует результат в KML для использования в AlpineQuest.

---

## Возможности

- Ввод двух точек области:
  - `NW` — верхняя левая точка;
  - `SE` — нижняя правая точка.
- Координаты в метрах:
  - `X` — северинг;
  - `Y` — восток.
- Автоматическое преобразование координат СК-42 / Гаусс-Крюгер в WGS84 для KML.
- Построение сеток:
  - `1000x1000`;
  - `100x100`;
  - `100x100` внутри каждого большого квадрата `1000x1000`.
- Нумерация больших квадратов `1000x1000` змейкой или стандартным порядком.
- Переименование больших квадратов `1000x1000` по отдельности без изменения номеров `100x100` внутри них.
- Настройка KML-стиля:
  - цвет и толщина линий `1000x1000`;
  - цвет и толщина линий `100x100`;
  - заливка больших квадратов `1000x1000`;
  - заливка всех квадратов `100x100` одним цветом;
  - один цвет, палитра по номерам или пользовательская палитра;
  - прозрачность заливок.
- Автоматический 2D-предпросмотр без карты:
  - сетка `1000x1000` и выбранные цвета/заливки;
  - выбор большого квадрата кликом;
  - детализация `100x100` внутри выбранного квадрата;
  - предпросмотр нумерации `100x100`;
  - масштабирование колесом, кнопками и фокус на выбранном квадрате;
  - перемещение схемы перетаскиванием.
- Гибкая нумерация малых квадратов `100x100`:
  - обычная;
  - змейкой;
  - спираль от центра наружу;
  - спираль от края к центру;
  - по строкам;
  - по колонкам;
  - старт из углов `NW`, `NE`, `SW`, `SE`;
  - направление спирали по часовой стрелке или против часовой стрелки;
  - якорь центра для четных сеток, включая `10x10`.
- Экспорт:
  - один общий `.kml`;
  - `.zip` с отдельным `tile_###.kml` на каждый квадрат `1000x1000`;
  - отдельная вкладка `Экспорт` с выбором формата, папки, имени файла и сводкой перед запуском;
  - оценка количества KML-объектов и примерного размера результата;
  - предупреждения и блокировка для чрезмерно больших экспортов;
  - проверка свободного места перед записью;
  - отмена долгого экспорта без сохранения незавершенного файла.
- Сохранение проекта:
  - файл `.lgm.json` с координатами, настройками сетки, именами квадратов, KML-стилем и экспортом;
  - открытие сохраненного проекта без потери настроек;
  - встроенные пресеты для старого режима `v0.0.1`, схем нумерации и KML-стилей.

KML-стиль по умолчанию остается спокойным и читаемым: стандартные черные линии, без заливки. Если заливка `100x100` включена, она прописывается в каждом малом квадрате одним выбранным цветом и прозрачностью; отдельных настроек для каждого `100x100` нет.

## Интерфейс

Приложение использует PySide6 и сделано как Windows desktop-инструмент.

Основные зоны интерфейса:

- левая панель: координаты, параметры сетки, схема нумерации, имена квадратов, стиль KML;
- правая панель: вкладки `Предпросмотр`, `Проверка` и `Экспорт`;
- верхняя панель проекта: новый проект, открытие, сохранение, сохранение как и применение пресетов;
- нижняя панель: генерация, отмена экспорта, прогресс экспорта, открытие папки результата.

Настройки стиля KML вынесены в отдельный диалог, а на основном экране показывается короткое резюме текущих цветов, толщин и режима заливки.

Вкладка `Предпросмотр` обновляется автоматически после изменения координат, округления, схемы нумерации, палитры или пользовательских имен квадратов. Для больших областей плотные подписи скрываются, чтобы схема оставалась отзывчивой и не ломала интерфейс.

Вкладка `Экспорт` показывает текущий вариант экспорта, путь результата и краткую сводку: сколько файлов будет создано, что попадет внутрь ZIP-архива, сколько KML-объектов будет записано и какой размер результата ожидается примерно.

Файл проекта сохраняет рабочее состояние отдельно от KML/ZIP-экспорта. Пресеты применяются к текущим настройкам и не трогают введенные координаты; полный режим `v0.0.1` возвращает поведение ранней версии для сетки, нумерации, стиля и формата экспорта.

Экспорт выполняется в отдельном worker-потоке, чтобы окно не зависало на больших сетках. Во время генерации можно нажать `Отменить`; приложение удалит временный незавершенный файл и оставит прежний результат нетронутым.

## Координатная логика

Зона Гаусса-Крюгера определяется из координаты `Y`:

```python
zone = int(abs(y) // 1_000_000)
epsg = 28400 + zone
```

Преобразование выполняется через `pyproj`:

- исходная CRS: `EPSG:28400 + zone`;
- целевая CRS: `EPSG:4326`;
- `always_xy=True`.

Порядок важен:

- пользователь вводит `X, Y`;
- в `pyproj` передается `Y, X`;
- в KML записывается `lon,lat,0`.

```python
lon, lat = transformer.transform(y, x)
```

## Разработка

Установка зависимостей:

```powershell
uv sync --extra dev
```

Запуск из исходников:

```powershell
uv run limuzin-grid-manager
```

Тесты:

```powershell
uv run --extra dev pytest
```

Проверка компиляции:

```powershell
uv run --extra dev python -m compileall src tests
```

Сборка Windows EXE:

```powershell
.\build_exe_windows.bat
```

Готовый EXE создается здесь:

```text
dist/LIMUZIN_GRID_MANAGER.exe
```
