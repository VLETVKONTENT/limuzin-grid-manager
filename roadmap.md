# Roadmap до v1.0: LIMUZIN GRID MANAGER

Этот файл описывает путь развития LIMUZIN GRID MANAGER от ранних рабочих версий до стабильной версии `v1.0`.

Roadmap не является жестким законом: версии можно объединять, дробить или менять местами, если в процессе разработки станет понятно, что так удобнее. Но общий порядок важен: сначала расширяем базовую логику, затем добавляем предпросмотр, затем приводим интерфейс и экспорт к стабильному виду.

## Главные цели v1.0

- Гибкая нумерация малых квадратов `100x100` внутри каждого большого квадрата `1000x1000`.
- Несколько схем нумерации:
  - змейка из любого угла;
  - обычная построчная/поколонная нумерация из любого угла;
  - спираль от центра наружу;
  - спираль от края к центру;
  - дополнительные схемы, если они реально полезны в работе.
- Гибкая палитра KML:
  - возможность оставить заливку полностью отключенной;
  - выбор цвета линий;
  - выбор цвета заливки больших квадратов;
  - настройка прозрачности заливки больших квадратов;
  - заливка всех малых квадратов `100x100` одним общим цветом и прозрачностью, без отдельных настроек на каждый малый квадрат.
- Переименование больших квадратов `1000x1000` по отдельности:
  - стандартная нумерация остается по умолчанию;
  - пользователь может задать имя конкретному большому квадрату;
  - переименование большого квадрата не меняет нумерацию малых квадратов внутри него.
- Автоматический предпросмотр:
  - предпросмотр сетки;
  - предпросмотр нумерации;
  - предпросмотр выбранной цветовой схемы;
  - обновление без отдельной кнопки “обновить”.
- Развитый интерфейс:
  - новые функции не должны перегружать первый экран;
  - будущий экспорт лучше вынести в отдельную вкладку `Экспорт`;
  - настройки должны быть понятными и не раздражать при частом использовании.

## Базовые принципы

- Не ломать текущий экспорт `v0.0.1`: общий KML и ZIP по большим квадратам должны продолжать работать.
- Все новые схемы нумерации должны быть детерминированными: одинаковые настройки дают одинаковый KML.
- UI не должен требовать лишних действий: там, где возможно, статистика и предпросмотр обновляются автоматически.
- Логика нумерации, палитр и экспорта должна жить в `core`, а UI должен только собирать настройки и показывать результат.
- Любая новая опция экспорта должна быть покрыта тестами хотя бы на уровне ядра.
- Если функция усложняет интерфейс, ее нужно выносить в отдельную секцию, вкладку или диалог, а не добавлять все в одну большую панель.

## v0.1.0 — Движок схем нумерации `100x100`

Статус: реализовано.

Цель: заменить текущий простой boolean `snake_small` на расширяемую модель схем нумерации малых квадратов.

Что добавить:

- Новый enum/тип `SmallNumberingMode`.
- Новые параметры:
  - схема нумерации;
  - стартовый угол;
  - направление обхода, если оно применимо к выбранной схеме.
- Схемы первой очереди:
  - построчная нумерация слева направо;
  - построчная нумерация справа налево;
  - поколонная нумерация сверху вниз;
  - змейка по строкам;
  - змейка по колонкам.
- Стартовые углы:
  - `NW` — верхний левый;
  - `NE` — верхний правый;
  - `SW` — нижний левый;
  - `SE` — нижний правый.

UI:

- В блоке `Сетка` заменить чекбокс “Нумерация 100x100 змейкой” на компактный блок:
  - `Схема нумерации 100x100`;
  - `Стартовый угол`;
  - `Направление`.
- Для текущего поведения `v0.0.1` выставить дефолт: змейка по строкам, старт `NW`.

Критерии готовности:

- Старый сценарий “змейка сверху слева” дает тот же результат, что и `v0.0.1`.
- Все стартовые углы работают.
- Есть unit-тесты на матрицу `10x10`.

## v0.2.0 — Спиральная нумерация

Статус: реализовано.

Цель: добавить схемы, которые невозможно удобно описать одной змейкой.

Что добавить:

- `Спираль от центра наружу`.
- `Спираль от края к центру`.
- Настройку направления спирали:
  - по часовой стрелке;
  - против часовой стрелки.
- Правило для центра сетки `10x10`, так как у нее нет одной центральной клетки:
  - выбрать один из четырех центральных квадратов как стартовый якорь;
  - дефолтный якорь: верхний левый из центральных четырех.

Возможные дополнительные схемы для оценки:

- Диагональная нумерация.
- Нумерация “кольцами”.
- Нумерация по квадрантам.
- Ручная карта номеров для специальных задач.

UI:

- Для спирали показывать только релевантные настройки: центр/край, направление, якорь центра.
- Не показывать настройки, которые не относятся к выбранной схеме.

Критерии готовности:

- Спиральные схемы покрыты unit-тестами.
- Для `10x10` нет дубликатов и пропущенных номеров.
- Предыдущие схемы из `v0.1.0` не ломаются.

## v0.3.0 — Переименование больших квадратов

Статус: реализовано.

Цель: дать пользователю возможность переименовывать только большие квадраты `1000x1000`.

Что добавить:

- Модель имен больших квадратов:
  - ключ: стандартный номер большого квадрата;
  - значение: пользовательское имя;
  - пустое значение означает стандартное имя.
- В KML пользовательское имя влияет только на:
  - имя папки большого квадрата;
  - имя большого полигона, если это удобно для AlpineQuest.
- Нумерация малых квадратов `100x100` внутри большого квадрата не меняется.

UI:

- Добавить отдельную панель/диалог `Имена квадратов`.
- Показать таблицу больших квадратов:
  - стандартный номер;
  - пользовательское имя;
  - кнопка сброса имени.
- Не заставлять пользователя переименовывать все квадраты.

Критерии готовности:

- Если пользователь не менял имена, экспорт полностью совпадает по логике со стандартной нумерацией.
- Если пользователь переименовал квадрат `005`, это не влияет на номера `100x100` внутри него.
- Есть тест на KML-имя большого квадрата.

## v0.4.0 — Палитры, линии и заливка больших квадратов

Статус: реализовано.

Цель: вернуть цветовые возможности уже в управляемом виде, не ломая правило “заливки может не быть”.

Что добавить:

- Настройки линий:
  - цвет линии больших квадратов;
  - цвет линии малых квадратов;
  - толщина линии больших квадратов;
  - толщина линии малых квадратов.
- Настройки заливки больших квадратов:
  - заливка отключена;
  - один цвет для всех больших квадратов;
  - палитра по номерам больших квадратов;
  - пользовательская палитра.
- Прозрачность заливки больших квадратов:
  - `0%` — без видимой заливки;
  - `1..100%` — видимая заливка;
  - важно: в KML корректно преобразовывать прозрачность в alpha-канал.
- Заливка `100x100` работает через общий цвет и прозрачность для всех малых квадратов; отдельные настройки на каждый `100x100` не используются.

Дефолт:

- Как в `v0.0.1`: черные линии, заливка отключена.

UI:

- Не добавлять все настройки палитры в основной экран.
- Сделать отдельный блок или диалог `Стиль KML`.
- Показывать краткое резюме стиля на основном экране.

Критерии готовности:

- Можно получить KML полностью без заливки.
- Можно включить заливку больших квадратов и/или заливку всех малых квадратов `100x100`.
- Малые квадраты получают заливку одним общим цветом и прозрачностью, без отдельной настройки каждого квадрата.
- Есть тесты на KML `LineStyle`, `PolyStyle`, alpha, заливку `1000x1000` и заливку `100x100`.

## v0.5.0 — Предпросмотр сетки и нумерации

Статус: реализовано.

Цель: пользователь должен видеть, как будет выглядеть сетка и нумерация до экспорта.

Что добавить:

- 2D-предпросмотр без карты:
  - большие квадраты;
  - малые квадраты внутри выбранного большого квадрата;
  - номера квадратов;
  - направление нумерации;
  - выбранные цвета линий и заливки больших квадратов.
- Автоматическое обновление предпросмотра при изменении:
  - координат;
  - округления;
  - схемы нумерации;
  - стартового угла;
  - палитры;
  - переименования большого квадрата.

UI:

- Добавить отдельную область предпросмотра справа или отдельную вкладку `Предпросмотр`.
- Для больших областей не пытаться рисовать все подписи, если это становится нечитаемо.
- Дать масштабирование и перемещение предпросмотра.
- Выбранный большой квадрат должен быть виден и связан с таблицей переименования.

Критерии готовности:

- Изменение схемы нумерации сразу видно без кнопки “обновить”.
- Предпросмотр не зависает на больших сетках.
- При невозможном вводе предпросмотр показывает понятную ошибку или пустое состояние.

## v0.6.0 — Отдельная вкладка `Экспорт`

Статус: реализовано.

Цель: подготовить интерфейс к большому количеству экспортных вариантов.

Что добавить:

- Вкладку `Экспорт`.
- Внутри вкладки:
  - выбор формата;
  - выбор имени файла;
  - выбор папки;
  - режим один KML / ZIP;
  - будущие пресеты экспорта;
  - краткая сводка: сколько файлов будет создано.
- Основной экран оставить для координат, сетки, предпросмотра и статистики.

Возможные будущие экспортные направления:

- KML общий.
- ZIP по большим квадратам.
- Только большие квадраты.
- Только малые квадраты.
- Экспорт таблицы номеров.
- Экспорт настроек проекта.

Критерии готовности:

- Текущие KML/ZIP сценарии не ломаются.
- Экспортная вкладка не перегружает основной экран.
- Пользователь видит, что именно будет создано до запуска экспорта.

## v0.7.0 — Сохранение проекта и пресеты

Статус: реализовано.

Цель: пользователь должен иметь возможность вернуться к настройкам позже.

Что добавить:

- Файл проекта, например `.lgm.json`.
- Сохранение:
  - координат;
  - режимов сетки;
  - схем нумерации;
  - переименований больших квадратов;
  - палитры;
  - настроек экспорта.
- Пресеты:
  - дефолтный режим `v0.0.1`;
  - режимы нумерации;
  - стили KML.

UI:

- Меню или кнопки:
  - `Новый проект`;
  - `Открыть`;
  - `Сохранить`;
  - `Сохранить как`;
  - `Пресеты`.

Критерии готовности:

- Проект сохраняется и открывается без потери настроек.
- Старые файлы настроек можно мигрировать или показать понятную ошибку.

Статус: реализовано.

Цель: подготовить приложение к реальному тяжелому использованию.

Что улучшено:

- Ограничения и предупреждения для слишком больших сеток.
- Более точный прогресс экспорта.
- Отмена экспорта.
- Проверка свободного места перед большим ZIP.
- Ускорение предпросмотра для больших областей.
- Более подробные сообщения об ошибках.

Критерии готовности:

- Пользователь может отменить долгий экспорт.
- Ошибки координат и зон понятны без технических деталей.

Состав `v1.0`:

- Надежный расчет сеток `1000x1000` и `100x100`.
- Гибкая нумерация малых квадратов.
- Переименование больших квадратов.
- Управляемая палитра KML.
- Отсутствие заливки как полноценный штатный режим.
- Предпросмотр сетки и нумерации.
- Отдельная вкладка экспорта.
- Сохранение проектов и пресетов.
- Документация и история версий.
- Стабильная Windows EXE-сборка.

Критерии готовности:

- Все тесты проходят.
- EXE запускается на Windows.
- Экспорт проверен вручную.
- В GitHub есть release `v1.0.0`.
- `GRIDBASE.md`, `roadmap.md`, `versions/GRIDVERSIONS.md` обновлены.
